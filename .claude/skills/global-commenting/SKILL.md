## When to use this skill

- When deciding whether to add comments to code
- When writing PHPDoc blocks for PHP classes and methods
- When adding JSDoc comments to JavaScript/TypeScript functions
- When documenting complex algorithms or business logic
- When writing self-documenting code with clear naming
- When removing outdated or unnecessary comments
- When documenting API endpoints or public interfaces
- When explaining non-obvious implementation decisions
- When avoiding comments about temporary changes or fixes
